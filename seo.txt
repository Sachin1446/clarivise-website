2. CRITICAL SEO ISSUES FOR REACT SPAs (What Needs Fixing)
================================================================================

Since you're using a Single Page Application (SPA), these are CRITICAL:

❌ PRERENDERING/SSR ISSUE
   Problem: Search engines may not execute JavaScript fully.
   Solution: Generate static HTML or use prerendering.

   OPTIONS:
   a) Add prerendering with vite-plugin-ssr or vite-ssg
   b) Use a service like prerender.io
   c) Convert to Next.js for built-in SSR

❌ DYNAMIC META TAGS PER PAGE
   Problem: All pages share the same index.html meta tags.
   Solution: Use react-helmet-async for page-specific meta tags.

   INSTALL:
   npm install react-helmet-async

   USAGE:
   import { Helmet } from 'react-helmet-async';

   function GyaniBuddy() {
     return (
       <>
         <Helmet>
           <title>Gyani Buddy - AI Learning Assistant | Clarivise</title>
           <meta name="description" content="Gyani Buddy helps students..." />
           <link rel="canonical" href="https://clarivise.in/gyani-buddy" />
         </Helmet>
         <Component />
       </>
     );
   }

❌ SITEMAP.XML
   Problem: No sitemap for search engines to crawl.
   Solution: Generate sitemap.xml and robots.txt.

   CREATE: public/sitemap.xml
   <?xml version="1.0" encoding="UTF-8"?>
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <url>
       <loc>https://clarivise.in/</loc>
       <lastmod>2024-01-01</lastmod>
       <changefreq>weekly</changefreq>
       <priority>1.0</priority>
     </url>
     <url>
       <loc>https://clarivise.in/gyani-buddy</loc>
       <lastmod>2024-01-01</lastmod>
       <changefreq>monthly</changefreq>
       <priority>0.8</priority>
     </url>
   </urlset>

   CREATE: public/robots.txt
   User-agent: *
   Allow: /
   Sitemap: https://clarivise.in/sitemap.xml

❌ IMAGE ALT TEXT
   Problem: Images may lack descriptive alt attributes.
   Solution: Add meaningful alt text to all images.

   BAD:  <img src="ai.jpg" alt="image" />
   GOOD: <img src="ai.jpg" alt="AI-powered business automation dashboard" />

❌ SEMANTIC HTML
   Problem: Using divs instead of semantic HTML5 tags.
   Solution: Use <header>, <main>, <section>, <article>, <footer>, <nav>

❌ BROKEN LINKS
   Problem: Internal links may lead to 404s.
   Solution: Regularly audit with tools like Screaming Frog.

3. ON-PAGE SEO OPTIMIZATION
================================================================================

A. TITLE TAGS & META DESCRIPTIONS
---------------------------------
   - Keep titles under 60 characters
   - Keep descriptions under 160 characters
   - Include primary keyword at the beginning
   - Make them unique for every page
   - Include your brand name

   EXAMPLES:
   Home: "AI Solutions for Business | Clarivise"
   Gyani Buddy: "Gyani Buddy - AI Learning Platform | Clarivise"
   About: "About Clarivise - AI Innovation Company"

B. HEADER TAGS (H1, H2, H3)
---------------------------
   - Use only ONE H1 per page (currently done well)
   - H1 should include your primary keyword
   - Use H2s for main sections
   - Use H3s for subsections
   - Create a logical hierarchy

C. KEYWORD OPTIMIZATION
----------------------
   PRIMARY KEYWORDS (target these):
   - AI solutions India
   - Business automation AI
   - AI consulting services
   - Machine learning solutions
   - Gyani Buddy

   LONG-TAIL KEYWORDS:
   - "AI solutions for small businesses in India"
   - "affordable AI automation services"
   - "AI trading insights platform"
   - "educational AI tools India"

   KEYWORD PLACEMENT:
   - URL (e.g., /ai-solutions not /services)
   - Title tag
   - H1 heading
   - First paragraph
   - Meta description
   - Image alt text

D. CONTENT OPTIMIZATION
----------------------
   - Write for humans first, search engines second
   - Aim for 1000+ words on important pages
   - Use keywords naturally (don't stuff)
   - Include related terms (LSI keywords)
   - Add fresh content regularly (blog)

E. INTERNAL LINKING
------------------
   - Link between related pages
   - Use descriptive anchor text
   - Create a clear site structure
   - Add breadcrumb navigation

   EXAMPLE:
   "Learn more about our <a href="/ai-solutions">AI solutions</a>"

F. IMAGE OPTIMIZATION
---------------------
   - Use descriptive file names: ai-dashboard.jpg not IMG_001.jpg
   - Add alt text to every image
   - Compress images (use WebP format)
   - Implement lazy loading
   - Add width and height attributes

   IMPLEMENTATION:
   <img
     src="/images/ai-solution.webp"
     alt="AI-powered business dashboard showing analytics"
     width="800"
     height="600"
     loading="lazy"
   />

4. TECHNICAL SEO
================================================================================

A. PAGE SPEED OPTIMIZATION
-------------------------
   Currently using Vite (good for speed). Also do:

   ✓ Code splitting (Vite does this automatically)
   ✓ Lazy load routes
   ✓ Optimize images (use WebP)
   ✓ Minimize JavaScript
   ✓ Use CDN for assets
   ✓ Enable gzip/brotli compression

   TOOLS:
   - Google PageSpeed Insights
   - GTmetrix
   - Web Vitals Chrome Extension

B. CORE WEB VITALS
-----------------
   Monitor these metrics:
   - LCP (Largest Contentful Paint): < 2.5s
   - FID (First Input Delay): < 100ms
   - CLS (Cumulative Layout Shift): < 0.1

C. MOBILE RESPONSIVENESS
-----------------------
   You have viewport meta tag (good).
   Also ensure:
   - Touch targets are at least 48x48px
   - Text is readable without zooming
   - No horizontal scrolling
   - Test on real devices

D. SSL/HTTPS
------------
   Ensure HTTPS is enabled (certificates valid).
   Redirect HTTP to HTTPS.

E. URL STRUCTURE
---------------
   GOOD:
   - clarivise.in/ai-solutions
   - clarivise.in/gyani-buddy

   BAD:
   - clarivise.in/page?id=123
   - clarivise.in/services#ai

   Use hyphens, not underscores.
   Keep URLs lowercase.

F. BROKEN LINKS & 404s
---------------------
   - Create custom 404 page
   - Fix broken internal links
   - Redirect deleted pages (301)

G. SCHEMA MARKUP (ENHANCE EXISTING)
----------------------------------
   Add more specific schemas:

   FOR SERVICE PAGES:
   {
     "@context": "https://schema.org",
     "@type": "Service",
     "name": "AI Business Solutions",
     "provider": {
       "@type": "Organization",
       "name": "Clarivise"
     },
     "areaServed": "India",
     "hasOfferCatalog": {...}
   }

   FOR FAQ SECTIONS:
   {
     "@context": "https://schema.org",
     "@type": "FAQPage",
     "mainEntity": [{
       "@type": "Question",
       "name": "What is Clarivise?",
       "acceptedAnswer": {
         "@type": "Answer",
         "text": "Clarivise is..."
       }
     }]
   }
